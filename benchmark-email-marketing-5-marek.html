Każda pozycja ma diagnozę na naszych danych + dwa gotowe prompty: do Claude Code (wykonanie przez Klaviyo/Shopify MCP) i do Claude (design) (szablon maila). Kliknij „kopiuj" i wklej.

01
Priorytet · źródło: Seed + wszystkie 5

Welcome series zamiast pojedynczego maila

Diagnoza: nasz welcome (flow STdfpu) to JEDEN mail na generycznym szablonie „[Trimmed] Newsletter #1", a generuje ~273 tys. PLN/rok (≈81% przychodu z flow, open 61% / CTR 17%). Seed wysyła 5 maili w 4 dni. Każdy dobrze zrobiony kolejny mail to niemal czysty zysk.

Prompt → Claude Code
Użyj klaviyo MCP. Przeanalizuj welcome flow STdfpu (get_flow + get_flow_report za 12 mies., conversion metric Placed Order R6aTMS) i wypisz obecną strukturę oraz wyniki.
Zaproponuj serię 4 maili: 1) powitanie + kod (jest), 2) dowód społeczny + edukacja produktu, 3) plain-text od założyciela, 4) last-chance 24h przed wygaśnięciem kodu. Timing 0h / +24h / +72h / +120h.
Dla każdego: temat PL (3 warianty), preview, treść. Wygeneruj 3 nowe szablony przez create_email_template i daj instrukcję złożenia serii w Klaviyo UI. Warunek wejścia MUSI zawierać consent „subscribed". Pokaż plan, zanim coś utworzysz.
Prompt → Claude (design)
Zaprojektuj responsywny (mobile-first, max 600px) szablon HTML maila powitalnego GENACTIV:
- kolory i logo marki, kod rabatowy jako TEKST w ramce (nie grafika), jeden duży CTA „Odbierz rabat",
- sekcja „Czego się spodziewać" (3 punkty) + pasek zaufania (opinie / gwarancja),
- stopka: adres firmy, wypis, disclaimer suplementowy EU.
Zwróć czysty HTML z inline CSS, gotowy do wklejenia w edytorze Code w Klaviyo.

Najwyższy zwrot · brakujący money-flow

Replenishment — uzupełnienie zapasu

Diagnoza: nie mamy flow uzupełnienia zapasu, a produkt jest konsumowalny (AOV ~285 PLN, model powtarzalny). Draft „Post buy_refreshments" (WgXWvU) leży nieuruchomiony. Dla suplementów to zwykle #1 brakujący przychód z automatyzacji.

Prompt → Claude Code
Cel: uruchomić flow replenishment — brakujący money-flow #1.
1) Przez shopify MCP (get-customer-orders / get-orders) policz medianę odstępu dni między kolejnymi zamówieniami tego samego klienta dla top produktów = realny cykl zużycia.
2) Przez klaviyo MCP sprawdź draft WgXWvU i raport, czy coś już działa.
3) Zaproponuj flow: trigger Placed Order, opóźnienie = cykl zużycia minus 7 dni (osobno per produkt, np. 30/45/60 dni), 2 maile: „Czas uzupełnić zapas" + przypomnienie z linkiem 1-click reorder.
Napisz copy PL, utwórz szablony. Pokaż wyliczony interwał z danych PRZED budową.
Prompt → Claude (design)
Zaprojektuj mail „Czas uzupełnić zapas" GENACTIV (HTML, mobile-first):

05
Naprawa · źródło: porównanie z welcome

Browse abandonment — diagnoza 20% open rate

Diagnoza: flow Browse_test (VfCSbb) ma open rate ~20% — połowa zdrowego (welcome ma 61%). To sygnał problemu z deliverability, zbyt szerokim triggerem albo segmentem niezaangażowanych.

Prompt → Claude Code
Użyj klaviyo MCP. Zdiagnozuj flow Browse_test (VfCSbb): open ~20% vs welcome 61%.
Sprawdź: filtr wejścia (czy nie łapie niezaangażowanych), warunek consent/subscribed, smart sending, bounce/spam vs inne flow, oraz czy temat/preview nie wyglądają na spam.
Zwróć listę przyczyn w kolejności prawdopodobieństwa + konkretne poprawki (np. zawęź trigger do zaangażowanych z ostatnich 30–60 dni) i 3 nowe tematy maili.
Prompt → Claude (design)
Przeprojektuj mail browse-abandonment na lekki, szybko ładujący się układ (mniej grafik = lepsza dostarczalność):
1 oglądany produkt + 2–3 rekomendacje, jeden CTA „Wróć i zobacz", krótki nagłówek. Inline CSS, mobile-first.
